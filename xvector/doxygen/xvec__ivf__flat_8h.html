Detailed Description

IVF-Flat (Inverted File with Flat vectors) index interface.

This header provides the API for loading, saving, and searching IVF-Flat indices. IVF-Flat uses inverted file indexing with exact distance computation within clusters, providing a good balance of speed and accuracy.

Loads an IVF-Flat index from an already-loaded Faiss index in memory.

This function allows loading an IVF-Flat index from a faiss::Index* that has already been loaded into memory, avoiding the need to load from file again. The Faiss index must be of type IndexIVFFlat or a compatible type.

Note
The caller retains ownership of the Faiss index and is responsible for managing its lifecycle. The created XVector index does not take ownership of the Faiss index.
Parameters
[in]contextExecution context for device operations
[in]faissIndexPointer to an already-loaded Faiss index (faiss::Index*). Must be a valid IVF-Flat index. Internally validated via dynamic_cast to faiss::IndexIVFFlat; returns XVEC_ERROR_IO if the cast fails.
[out]outIndexHandlePointer to receive the handle to the loaded index. The caller is responsible for destroying this index using xvecIvfFlatDestroyIndex.
Returns
XVEC_SUCCESS on success, XVEC_ERROR_IO if faissIndex is not a valid IndexIVFFlat, or other error code otherwise

Performs an IVF-Flat fine search only (skips coarse search).

This function performs only the fine search phase using pre-computed nearest cluster IDs. Use this when the coarse search has already been performed externally (e.g., by FAISS) and only the fine search needs to be offloaded to XVector.

The typical use case is XFaiss integration where:

  1. FAISS performs coarse search on CPU to find nearest clusters
  2. XVector performs fine search on accelerator using the cluster IDs from step 1
Parameters
[in]contextExecution context for device operations
[in]indexHandleHandle to the IVF-Flat index containing inverted lists
[in]fineSearchQueryFine search query containing query vectors and pre-computed nearest cluster IDs (from external coarse search)
[out]resultsBufferHandleBuffer to receive search results. Must be pre-allocated with size: queryVectorCount * topK * sizeof(xvecIvfFlatSearchResultFloat32)
[out]outTimingPointer to receive timing information (can be NULL)
Returns
XVEC_SUCCESS on success, error code otherwise
