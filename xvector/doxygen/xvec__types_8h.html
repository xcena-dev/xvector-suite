Detailed Description

Core types, enums, and structures for the XVector library.

This header defines the fundamental types used throughout the XVector library, including status codes, distance metrics, opaque handles, and search query/result structures.

typedef struct xvecIvfFlatFineSearchQueryFloat32 xvecIvfFlatFineSearchQueryFloat32

Search query parameters for IVF-Flat fine search only.

This structure is used when the coarse search has already been performed externally (e.g., by FAISS) and only the fine search needs to be executed on XVector. Unlike xvecIvfFlatSearchQueryFloat32, this structure takes pre-computed nearest cluster IDs instead of numProbes.

IVF-RaBitQ search query structure.

Search query parameters for IVF-RaBitQ index search.

Contains all parameters needed to perform a search on an IVF-RaBitQ index. For query quantization settings, zero-initialization provides sensible defaults (quantization disabled). Enable query quantization for faster search at the cost of some accuracy.

Enumerator

Inner product (dot product).

Higher values indicate greater similarity.

Euclidean (L2) distance.

Lower values indicate greater similarity.

Cosine similarity.

Higher values indicate greater similarity.
