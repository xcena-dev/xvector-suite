Detailed Description

Execution context management for the XVector library.

This header provides the API for creating, configuring, and destroying execution contexts. An execution context encapsulates the device handle, module path, and runtime properties required for XVector operations.

Create an execution context.

Initializes a new execution context associated with the specified device and module path. The created context manages resources required for subsequent operations such as buffer allocations and kernel execution.

The returned context handle (xvecExecutionContextTag) must be released using xvecDestroyContext when it is no longer needed.

Get a property from the execution context.

Retrieves a named property value from the execution context as a string. If the property exists, its value will be copied to the provided buffer. The buffer must be large enough to hold the property value and null terminator.

To query the required buffer size, pass NULL for value and 0 for valueSize. The required size (including null terminator) will be written to outRequiredSize.

Parameters
[in]contextExecution context handle. Must not be NULL.
[in]keyProperty name. Must be a null-terminated string. Must not be NULL.
[out]valueBuffer to receive property value. May be NULL to query required size.
[in]valueSizeSize of the value buffer in bytes.
[out]outRequiredSizeOptional. If not NULL, receives the required buffer size including null terminator.
Returns
xvecStatus
  • XVEC_SUCCESS: Property retrieved successfully (or size query succeeded when value is NULL)
  • XVEC_ERROR_INVALID_PARAMETER: Invalid parameters (NULL context or key)
  • XVEC_ERROR_ENTRY_NOT_FOUND: Property with the given key does not exist
  • XVEC_ERROR_BUFFER_TOO_SMALL: Provided buffer is too small for the property value
  • XVEC_ERROR_UNKNOWN: Other errors

◆ xvecResetVpeContext()

xvecStatus xvecResetVpeContext ( xvecExecutionContextTag *  context)

Reset the VPE (Vector Process Engine) SRAM allocator state.

Clears the SRAM buffer allocation state across all subsystems on the device. This must be called before executing kernels that use VpeContext to ensure a clean allocator state.

The number of subsystems to reset is determined automatically from the available Sub count reported by the PXL runtime.

Parameters
[in]contextExecution context handle. Must not be NULL.
Returns
xvecStatus
  • XVEC_SUCCESS: VPE state reset successfully
  • XVEC_ERROR_INVALID_PARAMETER: context is NULL
Warning
This function must NOT be called while device kernels are actively using VpeContext buffers. Doing so will corrupt allocator state.
xvecExecutionContextTag* context;
xvecCreateContext(&context, 0, "module.mubin");
// Reset VPE before running kernels
xvecResetVpeContext(context);

Set a property on the execution context.

Sets a named property with a string value on the execution context. The caller is responsible for converting the value to the appropriate type when reading the property. Common properties include:

  • "numSub": Number of sub-tasks
  • "taskCount": Task count for parallel execution
  • "deviceLogLevel": Device log level ("0"=NONE, "1"=ERROR, "2"=WARN, "3"=INFO, "4"=DEBUG, "5"=TRACE)
Parameters
[in]contextExecution context handle. Must not be NULL.
[in]keyProperty name. Must be a null-terminated string. Must not be NULL.
[in]valueProperty value as a string. Must be a null-terminated string. Must not be NULL.
Returns
xvecStatus
