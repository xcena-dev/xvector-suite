Detailed Description

This is a convenience header that includes all public xarith components.

Users can include this single header to get access to all functionality.

The xarith library is a device-side C++ library designed for XCENA's MX1 accelerator device. It works alongside the MU library to build high-performance MU kernels that run on the MX1 device's VPE (Vector Processing Engine). The library provides a high-level interface for Vector Process Engine (VPE) operations, including buffer management, data transfer between DRAM and SRAM, and various vector operations (dot product, arithmetic, reduction, comparison).

The xarith library offers:

  • Automatic SRAM buffer allocation and lifecycle management via VpeContext
  • Synchronous vector operations optimized for hardware acceleration
  • Flexible VPE ID assignment strategies for load balancing
  • Thread-safe buffer operations with internal locking

enum class mu::vdma::VpeIdStrategy
strong

VPE ID calculation strategy for load balancing.

The VpeIdStrategy determines how VPE (Vector Process Engine) IDs are assigned to distribute workload across available hardware resources. Each strategy optimizes for different parallelization patterns:

ByThreadId (Default):

  • Calculation: vpeId = threadId & 0x1
  • Distributes threads within the same MU (Memory Unit) across different VPEs
  • Best for: Thread-level parallelism within a single cluster
  • Use case: Multiple CPU threads processing independent vectors on the same MU

ByClusterId:

  • Calculation: vpeId = clusterId & 0x1
  • Distributes MUs from different clusters across different VPEs
  • Best for: Cluster-level parallelism with multiple MUs
  • Use case: Multi-cluster workloads where each cluster should use a dedicated VPE
Note
Choose ByThreadId when parallelizing across CPU threads on a single MU. Choose ByClusterId when distributing work across multiple clusters/MUs.
The current implementation uses a simple modulo-2 strategy (& 0x1), which assumes 2 VPEs per system. Future versions may support more VPEs.
